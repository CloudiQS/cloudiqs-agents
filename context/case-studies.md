# Case Studies

Use these in email outreach, sales conversations, and agent-generated content. Always match the case study vertical to the prospect's industry.

---

## Voly Group — Monolith to ECS (MSP + Migration)

**Sector:** Fintech / Payments
**Size:** 80 employees
**Challenge:** Monolithic application on-prem, scaling bottlenecks during peak transaction periods, rising hardware costs.
**What we did:** Re-architected the application into containerised microservices, migrated to ECS Fargate on AWS, implemented Aurora PostgreSQL, set up CloudWatch dashboards and alerting.
**Outcome:** 60% reduction in infrastructure costs. Zero-downtime migration. Peak throughput increased 4x. Team now deploys daily instead of quarterly.
**Timeline:** 14 weeks
**Funding:** MAP-funded migration

**Use in outreach for:** Fintech, payments, any company with scaling pain or legacy monolith.

## UK Fintech — SOC2 Type II on AWS (Security + Compliance)

**Sector:** Financial services
**Size:** 120 employees, FCA regulated
**Challenge:** Needed SOC2 Type II certification to close enterprise contracts. Existing AWS environment had no audit trail, no access controls, and no incident response process.
**What we did:** Implemented GuardDuty, Config, CloudTrail, Security Hub, IAM least-privilege review, VPC flow logs, WAF, and automated compliance reporting. Prepared all evidence for audit.
**Outcome:** Passed SOC2 Type II on first audit. Closed 3 enterprise contracts worth £1.2M within 60 days of certification.
**Timeline:** 6 months
**Funding:** Well-Architected Review (WAR-funded)

**Use in outreach for:** Fintech, insurtech, any FCA-regulated or SOC2-seeking company.

## UK Manufacturing — SAP Migration (Migration)

**Sector:** Manufacturing / Industrial
**Size:** 250 employees, 3 UK sites
**Challenge:** On-prem SAP on ageing hardware, end-of-life in 18 months. IT team had no AWS experience. Business could not afford downtime during cutover.
**What we did:** Assessed SAP landscape, designed AWS architecture (EC2, EFS, RDS), ran parallel environment for 6 weeks, executed zero-downtime cutover on a Sunday with rollback plan in place.
**Outcome:** SAP on AWS in 14 weeks. Zero downtime. Annual infrastructure cost reduced from £180K to £65K. IT team trained and self-sufficient on Day 1.
**Timeline:** 14 weeks
**Funding:** MAP (£45K AWS credit applied)

**Use in outreach for:** Manufacturing, ERP users, VMware exits, any company with on-prem hardware end-of-life.

## UK Media Company — FSx for Windows (Storage Migration)

**Sector:** Media / Creative / Production
**Size:** 60 employees
**Challenge:** 200TB on-prem NAS, ageing hardware, expensive maintenance contract, staff working from 3 locations with poor remote access performance.
**What we did:** Migrated 200TB to FSx for Windows File Server, configured AD integration, set up DataSync for initial migration and ongoing sync, implemented backup policy with 90-day retention.
**Outcome:** 99.99% availability since migration (18 months). Staff access files at full speed from any location. NAS maintenance contract cancelled: £40K/year saving. Disaster recovery now automated.
**Timeline:** 8 weeks
**Funding:** MAP-funded

**Use in outreach for:** Media, creative agencies, architecture firms, any company with on-prem file storage or remote working challenges.

## How to use case studies in outreach

- One case study per email maximum
- Match the vertical (do not send the media case study to a fintech prospect)
- Lead with the outcome, not the technology: "60% cost reduction" not "ECS Fargate migration"
- Never name the customer unless you have explicit permission (these are anonymised)
- If the prospect mentions a specific challenge, match the case study to that challenge, not to their industry
